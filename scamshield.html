Scam Shield

AI-Powered Phishing Detection Browser Extension

Machine Learning TensorFlow Python NLP Firefox Extension LSTM

Project Overview

Scam Shield is my Honours Project (Graded as an A), the brief was to develop an ML model to detect phshing emails and deploy it into the browser for user interaction.

An LSTM was decided upon due to it's proven ability to reliably detect textual patterns, ideal for phishing emails. Development was undertaken in the TensorFlow framework, due to it's easy conversion to TensorFlow.js for future deployment.

Core Concepts

Machine Learning Architecture

The project utilises Long Short-Term Memory (LSTM) networks, a specialised type of recurrent neural network particularly effective for sequential data processing. This architecture is ideal for analyzing URL patterns and text content that characterize phishing attempts.

  • LSTM networks for sequential pattern recognition
  • Natural Language Processing for content analysis
  • Feature extraction from page content
  • Real-time classification with sub-second response times

Privacy-First Design

All processing occurs locally on the user's device, ensuring that sensitive browsing data never leaves their own browser. This approach addresses critical privacy concerns while maintaining the effectiveness of the security solution.

Development & Testing

Model Development

The LSTM model was developed using TensorFlow and trained on a comprehensive dataset of legitimate and phishing websites. The development process involved extensive experimentation with different architectures and hyperparameters.

Deployment & Integration

Firefox Extension Development

The trained model was integrated into a Firefox browser extension using TensorFlow.js, allowing for seamless client-side execution without requiring external API calls or server dependencies.


Browser → Content Script → ML Model → User Alert

Technical Implementation

  • Model conversion to TensorFlow.js format
  • Web Scraping HTML for email body
  • Real-time content monitoring
  • Non-intrusive user notification system
  • Performance optimisation for browser compatibility

User Experience Design

The extension was designed to be as non-intrusive as possible, only alerting users when Scam Shield believes there is a high probability of an email being phishing (>80%). A banner is displayed beside sender information detailing probability for all emails.

User Testing & Results

Testing Methodology

Users were asked to detect phishing emails with and without the aid of Scam Shield, after which they were to rate their own percieved confidence of their ability to detect phishing attempts. SUS was used to evaluate the tools actual usability and impact on the user.

Impact Assessment

  • Participants were 10% more likely to spot phishing emails with Scam Shield.
  • The average participant's confidence in spotting phishing attempts increased by 15% when using Scam Shield.
  • Participants strongly agreed with postiive usability statements like "easy to use", while strongly disagreeing with negative statements such as "unnecessarirly complex".

Future Enhancements

I am planning on retraining the model on a newer dataset, and add the option for users to report emails as phishing. This will allow the model to remain useful against current phishing techniques.
